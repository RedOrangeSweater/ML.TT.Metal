## /curs.review — разбор чата и предложения по правилам/командам

Разобрать этот чат (и опционально сообщение после команды): что пошло не так → предложить правки в `.cursor/rules/`, `.cursor/commands/`, при необходимости в AGENTS.md / CLAUDE.md.

### Input

Опционально: подсказка («focus on git», «focus on rules») или сообщение для разбора. Пусто → разбор всего чата.

### Output

- **What went wrong**: буллеты (с короткой отсылкой к чату).
- **Proposed updates**: для каждого файла — точный фрагмент (добавить/заменить). Новое правило — имя файла + полный контент.

### Next step

Если пользователь принимает — применить через команду/скилл (не ad-hoc). Правки правил/команд — через `/curs.apply` или соответствующий workflow (`14_rules_feedback_loop`).

### Constraints

Только изменения, устраняющие выявленные ошибки. Одно правило на один тип ошибки.

### Task (внутренний)

1. Выявить: недопонимания, невыполненные запросы, правки пользователя, отсутствующие действия.
2. Идеал: после каждого запроса — полный корректный ответ без повторных уточнений.
3. Цели правок: `.cursor/rules/*`, `.cursor/commands/*`, при наличии AGENTS.md/CLAUDE.md. Короткие, конкретные правки.
